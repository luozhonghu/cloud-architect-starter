# 核心服务 - 网络服务（VPC/CDN/SLB）

## 一、网络服务概述

### 1.1 网络服务的重要性

网络服务是云基础设施的重要组成部分，决定了应用的访问速度、安全性、可靠性。

**核心网络服务：**
- **VPC（虚拟私有云）**：隔离的网络环境
- **SLB（负载均衡）**：流量分发，高可用
- **CDN（内容分发网络）**：加速访问
- **EIP（弹性公网IP）**：公网访问
- **NAT网关**：内网访问外网

### 1.2 网络架构层次

```
用户 → CDN → WAF → SLB → ECS → 数据库
        ↓
      NAT 网关
```

## 二、虚拟私有云（VPC）

### 2.1 VPC 概念

VPC（Virtual Private Cloud）是云上隔离的虚拟网络环境，类似于在数据中心中搭建的私有网络。

**核心特性：**
- **网络隔离**：不同 VPC 完全隔离
- **自定义网络**：自定义 IP 地址段、子网、路由表
- **安全控制**：安全组、网络ACL
- **互联能力**：专线、VPN 连接

### 2.2 VPC 架构组成

#### 专有网络（VPC）
- 顶层网络容器
- 定义 IP 地址段（CIDR）
- 一个账户可创建多个 VPC

#### 交换机（VSwitch）
- VPC 的子网
- 可用区级别的网络划分
- 每个 VSwitch 属于一个可用区

#### 路由表
- 控制网络流量路由
- 可配置路由规则
- 支持静态路由和动态路由

#### 网关
- **公网网关**：访问公网
- **NAT 网关**：SNAT/DNAT
- **VPN 网关**：VPN 连接

### 2.3 IP 地址规划

#### VPC CIDR 规划
- 推荐使用 10.0.0.0/8 或 172.16.0.0/12
- 避免与现有网络冲突
- 预留足够的地址空间

#### 子网规划
- **生产子网**：部署生产应用
- **开发子网**：部署开发测试应用
- **管理子网**：部署管理服务
- **数据库子网**：部署数据库

#### 示例规划

| 子网 | CIDR | 用途 | 可用区 |
|------|------|------|--------|
| prod-web | 10.0.1.0/24 | Web 应用 | AZ1 |
| prod-app | 10.0.2.0/24 | 应用服务 | AZ1 |
| prod-db | 10.0.3.0/24 | 数据库 | AZ2 |
| dev-all | 10.0.100.0/24 | 开发环境 | AZ1 |

### 2.4 安全组 vs 网络ACL

| 特性 | 安全组 | 网络ACL |
|------|--------|---------|
| 作用范围 | 实例级别 | 子网级别 |
| 规则类型 | 有状态 | 无状态 |
| 默认策略 | 拒绝所有入站，允许所有出站 | 允许所有入站出站 |
| 优先级 | 优先应用 | 优先级控制 |
| 适用场景 | 实例访问控制 | 子网级别控制 |

### 2.5 VPC 互联

#### 对等连接
- 同一区域的 VPC 互联
- 点对点连接
- 低延迟

#### 专线连接
- 高速、稳定
- 通过物理专线连接
- 适合混合云

#### VPN 连接
- 基于 IPsec VPN
- 通过公网加密连接
- 成本较低

#### 云企业网（CEN）
- 跨地域、跨账号 VPC 互联
- 统一管理
- 支持多路由表

## 三、负载均衡（SLB）

### 3.1 负载均衡概述

负载均衡将流量分发到多个后端服务器，提高应用可用性和性能。

**核心价值：**
- **高可用**：自动剔除故障实例
- **弹性伸缩**：根据负载自动扩缩容
- **会话保持**：保持用户会话
- **健康检查**：监控实例状态

### 3.2 负载均衡类型

#### 四层负载均衡（TCP/UDP）
- 基于传输层
- 性能高
- 支持自定义 TCP/UDP 端口
- 适合非 HTTP 协议

#### 七层负载均衡（HTTP/HTTPS）
- 基于应用层
- 支持 HTTP/HTTPS
- 支持 URL 重写、重定向
- 支持会话保持

### 3.3 负载均衡实例类型

#### 共享型实例
- 多租户共享
- 成本低
- 适合中小型业务

#### 性能保障型实例
- 独享资源
- 性能稳定
- 适合中大型业务

#### 高级型实例
- 超高性能
- 支持更多功能
- 适合大型业务

### 3.4 负载均衡算法

#### 轮询（Round Robin）
- 按顺序分配请求
- 适合服务器性能相同

#### 加权轮询
- 根据权重分配
- 适合服务器性能不同

#### 最小连接数
- 分配到连接数最少的服务器
- 适合长连接

#### 源地址哈希
- 根据源 IP 哈希
- 适合需要会话保持的场景

### 3.5 健康检查

#### 健康检查方式
- **TCP 检查**：检查 TCP 端口
- **HTTP 检查**：检查 HTTP 响应码
- **HTTPS 检查**：检查 HTTPS 响应码

#### 健康检查参数
- 检查间隔：2-5 秒
- 超时时间：2-10 秒
- 健康阈值：2-3 次
- 不健康阈值：2-3 次

### 3.6 会话保持

#### 四层会话保持
- 基于源 IP
- 保持时间可配置

#### 七层会话保持
- 基于 Cookie
- 支持植入 Cookie 或重写 Cookie

### 3.7 SSL/TLS 卸载

#### 证书管理
- 上传 SSL 证书
- 自动续期

#### 加密策略
- 选择加密套件
- TLS 版本控制

## 四、内容分发网络（CDN）

### 4.1 CDN 概述

CDN（Content Delivery Network）通过全球分布的节点，将内容缓存到离用户最近的节点，加速访问。

**核心价值：**
- **加速访问**：减少网络延迟
- **降低源站压力**：回源流量减少
- **提升用户体验**：加载速度提升
- **节约成本**：带宽成本降低

### 4.2 CDN 工作原理

```
用户请求 → DNS 解析 → 就近 CDN 节点 → 命中缓存 → 返回内容
                              ↓
                          未命中
                              ↓
                         回源站获取
```

### 4.3 加速类型

#### 静态加速
- 图片、CSS、JS 等静态资源
- 缓存时间长
- 加速效果明显

#### 动态加速
- API 接口、动态页面
- 路由优化
- 协议优化

#### 视频加速
- 点播、直播
- 分片传输
- 自适应码率

### 4.4 缓存配置

#### 缓存规则
- 按文件类型缓存
- 按路径缓存
- 按参数缓存

#### 缓存时间
- **静态资源**：1 年
- **API 接口**：不缓存
- **图片**：1 个月

#### 示例配置

| 路径 | 缓存时间 | 优先级 |
|------|---------|--------|
| /images/* | 30天 | 高 |
| /css/*.css | 1年 | 高 |
| /js/*.js | 1年 | 高 |
| /api/* | 不缓存 | 高 |
| /*.html | 1小时 | 中 |

### 4.5 回源配置

#### 回源协议
- HTTP
- HTTPS
- 协议跟随

#### 回源 HOST
- 源站域名
- CDN 加速域名

#### 回源头配置
- 自定义回源请求头
- 传递真实客户端 IP

### 4.6 HTTPS 配置

#### 证书管理
- 上传证书
- 自动续期

#### HTTPS 配置
- 强制 HTTPS
- HTTP 自动跳转 HTTPS

### 4.7 访问控制

#### Referer 防盗链
- 白名单/黑名单
- 空 Referer 处理

#### IP 黑白名单
- IP 访问控制
- 地域封禁

#### URL 鉴权
- 时间戳鉴权
- 鉴权密钥

### 4.8 性能优化

#### 预热
- 提前推送到节点
- 大流量活动前预热

#### 刷新
- 更新缓存
- 支持目录刷新和 URL 刷新

#### 优化策略
- 开启 Gzip 压缩
- 使用 HTTP/2
- 合并文件

## 五、弹性公网 IP（EIP）

### 5.1 EIP 概述

EIP（Elastic IP）是可以独立申请和释放的公网 IP 地址，可以绑定到云服务器、负载均衡等资源。

**特性：**
- 独立申请释放
- 可绑定到不同资源
- 支持带宽调整
- 支持按流量或按带宽计费

### 5.2 EIP 使用场景

#### 绑定云服务器
- 直接公网访问
- 替代传统公网 IP

#### 绑定负载均衡
- 公网访问负载均衡

#### 绑定 NAT 网关
- 内网访问外网

### 5.3 EIP 计费模式

#### 按使用流量
- 按实际流量计费
- 适合流量波动大
- 成本可控

#### 按固定带宽
- 按带宽峰值计费
- 适合流量稳定
- 长期使用成本优惠

## 六、NAT 网关

### 6.1 NAT 网关概述

NAT（Network Address Translation）网关提供源地址转换（SNAT）和目的地址转换（DNAT）功能。

**功能：**
- **SNAT**：内网访问外网
- **DNAT**：外网访问内网
- **端口转发**：端口映射

### 6.2 SNAT 配置

#### 场景
- 内网 ECS 访问外网
- 不为每个 ECS 分配公网 IP
- 节省公网 IP 资源

#### 配置示例
```bash
# 创建 SNAT 条目
- VPC: vpc-xxxxx
- 交换机: vsw-xxxxx (10.0.1.0/24)
- EIP: 1.2.3.4
- SNAT IP: 1.2.3.4
```

### 6.3 DNAT 配置

#### 场景
- 外网访问内网 ECS
- 端口转发
- 映射到指定内网 IP

#### 配置示例
```bash
# 创建 DNAT 条目
- 公网 IP: 1.2.3.4
- 公网端口: 80
- 内网 IP: 10.0.1.10
- 内网端口: 80
```

## 七、网络架构最佳实践

### 7.1 典型 Web 应用架构

```
                    用户
                     ↓
                    CDN
                     ↓
                   WAF
                     ↓
                   SLB (公网)
                     ↓
                ┌───┴───┐
                ↓       ↓
              ECS-1   ECS-2
                ↓       ↓
               RDS-主  RDS-从
```

### 7.2 多可用区架构

```
                用户
                 ↓
               SLB (跨可用区)
         ┌───────┴───────┐
         ↓               ↓
      AZ1              AZ2
         ↓               ↓
      VSwitch-1       VSwitch-2
         ↓               ↓
      ECS集群          ECS集群
         ↓               ↓
      RDS-主           RDS-从
```

### 7.3 安全架构

```
        公网
         ↓
       CDN
         ↓
       WAF
         ↓
       SLB
         ↓
    DMZ 子网 (Web)
         ↓
    应用子网 (App)
         ↓
    数据子网 (DB)
```

### 7.4 混合云架构

```
        用户
         ↓
        CDN
         ↓
       SLB
         ↓
    云上应用
         ↓
    VPN/专线
         ↓
    数据中心
```

## 八、网络监控与故障排查

### 8.1 关键指标

#### 网络指标
- 带宽使用率
- 流量峰值
- 包丢失率
- 延迟

#### SLB 指标
- QPS
- 响应时间
- 后端健康状态
- 连接数

#### CDN 指标
- 命中率
- 回源流量
- 延迟
- 错误率

### 8.2 常见故障排查

#### 访问慢
- 检查 CDN 命中率
- 检查带宽使用
- 检查后端响应时间

#### 无法访问
- 检查安全组规则
- 检查路由表配置
- 检查 DNS 解析

#### 丢包
- 检查带宽限制
- 检查网络配置
- 检查后端负载

## 九、成本优化

### 9.1 成本组成

```
网络成本 = EIP费用 + 带宽费用 + SLB费用 + CDN费用 + 其他费用
```

### 9.2 成本优化策略

#### CDN 优化
- 提高缓存命中率
- 使用流量包
- 按需选择加速类型

#### 带宽优化
- 使用 CDN 降低源站流量
- 压缩传输内容
- 选择合适计费方式

#### SLB 优化
- 选择合适的实例规格
- 使用按量付费（如果流量波动大）

### 9.3 计费实例（阿里云）

| 服务 | 规格 | 月成本 |
|------|------|--------|
| SLB | 共享型 | ¥300 |
| EIP | 5Mbps 带宽 | ¥300 |
| CDN | 流量 1TB | ¥500 |
| **总计** | | **¥1100** |

## 十、总结

网络服务是云基础设施的基础，合理设计网络架构、选择合适的网络服务，可以有效提升应用的性能、安全性和可靠性。

**关键要点：**
1. VPC 是网络隔离的基础
2. 负载均衡实现高可用和负载分担
3. CDN 加速访问，降低成本
4. NAT 网关节省公网 IP
5. 网络架构要考虑安全和扩展性
6. 监控和故障排查必不可少
