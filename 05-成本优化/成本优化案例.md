## 三、优化方案

### 3.1 计算资源优化

#### 方案 1：降低 ECS 规格

**优化前：**
- 规格：4C16G
- 数量：5 台
- 月成本：¥10,000

**优化后：**
- 规格：2C4G
- 数量：10 台
- 月成本：¥6,000

**效果：**
- 节省：¥4,000 (40%)
- 并发能力提升：从 5 台提升到 10 台
- CPU 使用率：从 25% 提升到 60%

#### 方案 2：使用自动伸缩

**配置：**
- 最小实例数：2 台
- 最大实例数：10 台
- 默认实例数：4 台
- 扩容条件：CPU > 70%，持续 5 分钟
- 缩容条件：CPU < 30%，持续 15 分钟

**成本估算：**
- 高峰期（8 小时）：8 台
- 平峰期（12 小时）：4 台
- 低峰期（4 小时）：2 台

```
日均成本 = (8 × ¥200 × 8 + 4 × ¥200 × 12 + 2 × ¥200 × 4) / 24
        = (12,800 + 9,600 + 1,600) / 24
        = ¥1,000

月成本 = ¥1,000 × 30 = ¥3,000
```

**效果：**
- 节省：¥7,000 (70%)

#### 方案 3：使用包年包月

**按量付费：**
- 月成本：¥6,000
- 年成本：¥72,000

**包年包月：**
- 月成本：¥4,000
- 年成本：¥48,000
- 折扣：7 折

**效果：**
- 节省：¥24,000/年 (33%)

### 3.2 存储资源优化

#### 方案 1：使用 OSS 生命周期管理

**优化前：**
- 全部标准存储：2TB
- 月成本：2,048 × ¥0.12 = ¥245

**优化后：**
- 标准存储：500GB (新数据)
- 低频存储：1,500GB (30 天后)

```
月成本 = (500 × ¥0.12) + (1,500 × ¥0.08)
       = ¥60 + ¥120
       = ¥180
```

**效果：**
- 节省：¥65 (27%)

#### 方案 2：清理无用数据

**清理项：**
- 过期日志：200GB
- 临时文件：100GB
- 冗余备份：100GB

**效果：**
- 节省空间：400GB
- 节省成本：400 × ¥0.12 = ¥48/月

### 3.3 网络资源优化

#### 方案 1：优化 CDN 配置

**优化前：**
- 未配置缓存规则
- CDN 流量：5TB
- 月成本：¥2,500

**优化后：**
- 配置缓存规则
- CDN 命中率：从 30% 提升到 70%
- 回源流量：5TB × 30% = 1.5TB

```
CDN 成本 = 5TB × ¥0.5 = ¥2,500
回源成本 = 1.5TB × ¥0.5 = ¥750
总成本 = ¥2,500 + ¥750 = ¥3,250

对比不使用 CDN：
回源成本 = 5TB × ¥0.5 = ¥2,500

节省 = ¥2,500 - ¥3,250 = -¥750（成本增加，但性能提升）
```

**实际效果：**
- 源站带宽从 100Mbps 降低到 30Mbps
- EIP 成本从 ¥2,000 降低到 ¥600
- 总节省：¥2,000 - ¥600 - ¥750 = ¥650

#### 方案 2：EIP 计费方式优化

**优化前：**
- 固定带宽：100Mbps
- 月成本：¥2,000

**优化后：**
- 按流量计费
- 实际流量：1TB
- 月成本：1,024 × ¥0.5 = ¥512

**效果：**
- 节省：¥1,488 (74%)

### 3.4 数据库资源优化

#### 步骤 2：配置自动伸缩

```bash
# 1. 创建伸缩组
aliyun ess CreateScalingGroup \
  --ScalingGroupName app-scaling \
  --MinSize 2 \
  --MaxSize 10 \
  --DefaultCooldown 300

# 2. 创建伸缩配置
aliyun ess CreateScalingConfiguration \
  --ScalingGroupName app-scaling \
  --ImageId img-xxxxx \
  --InstanceType ecs.g6.large

# 3. 创建伸缩规则
aliyun ess CreateScalingRule \
  --ScalingRuleName scale-up \
  --ScalingGroupName app-scaling \
  --AdjustmentType QuantityChangeInCapacity \
  --AdjustmentValue 2

# 4. 创建任务
aliyun ess CreateScheduledTask \
  --ScheduledTaskName scale-up-task \
  --ScalingRuleId scale-up \
  --LaunchTime "2024-01-01T08:00:00Z"
```

#### 步骤 3：OSS 生命周期管理

```python
import oss2

# 配置生命周期规则
auth = oss2.Auth('AccessKeyId', 'AccessKeySecret')
bucket = oss2.Bucket(auth, 'https://oss-cn-hangzhou.aliyuncs.com', 'my-bucket')

# 创建生命周期规则
lifecycle_rule = oss2.models.LifecycleRule(
    id='lifecycle_rule',
    prefix='data/',
    status='Enabled',
    expiration=oss2.models.LifecycleExpiration(days=365),
    transitions=[
        oss2.models.LifecycleTransition(days=30, storage_class=oss2.BUCKET_IA),
        oss2.models.LifecycleTransition(days=90, storage_class=oss2.BUCKET_ARCHIVE)
    ]
)

bucket.put_bucket_lifecycle([lifecycle_rule])
```
