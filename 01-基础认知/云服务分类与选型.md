# 云服务分类与选型

## 一、云服务分类体系

### 1.1 按服务层次分类

#### IaaS 层服务（基础设施即服务）

| 服务类别 | 典型产品 | 主要功能 |
|---------|---------|---------|
| 计算 | ECS/CVM/EC2 | 虚拟服务器 |
| 存储 | OSS/COS/S3 | 对象存储 |
| 数据库 | RDS | 托管数据库 |
| 网络 | VPC/SLB/CDN | 虚拟网络、负载均衡、内容分发 |
| 安全 | WAF/DDoS 防护 | Web应用防火墙、DDoS防护 |

**特点：**
- 用户拥有更多控制权
- 需要管理操作系统、中间件、应用
- 适合需要灵活控制基础设施的场景

#### PaaS 层服务（平台即服务）

| 服务类别 | 典型产品 | 主要功能 |
|---------|---------|---------|
| 应用平台 | 云引擎、Cloud Foundry | 应用托管平台 |
| 容器平台 | ACK/TKE/EKS | Kubernetes 托管服务 |
| 函数计算 | 函数计算/SCF/Lambda | Serverless 计算平台 |
| 消息队列 | Kafka/MQ/RocketMQ | 消息中间件 |
| 缓存 | Redis/Memcached | 分布式缓存 |

**特点：**
- 用户无需管理基础设施
- 关注应用开发和部署
- 适合快速开发和部署

#### SaaS 层服务（软件即服务）

| 服务类别 | 典型产品 | 主要功能 |
|---------|---------|---------|
| 协同办公 | 钉钉/企业微信/飞书 | 即时通讯、文档协作 |
| 邮件服务 | 企业邮箱/Outlook | 企业邮件系统 |
| CRM | Salesforce/销售易 | 客户关系管理 |
| ERP | 用友/金蝶 | 企业资源计划 |

**特点：**
- 开箱即用
- 无需任何技术管理
- 适合标准化业务场景

### 1.2 按功能领域分类

#### 计算类服务

| 服务名称 | 功能描述 | 适用场景 |
|---------|---------|---------|
| 云服务器（ECS） | 虚拟化计算实例 | 通用计算、Web 应用 |
| 裸金属服务器 | 物理服务器性能 | 高性能计算、数据库 |
| GPU 实例 | GPU 加速计算 | AI 训练、渲染、科学计算 |
| FPGA 实例 | FPGA 可编程硬件 | 加密解密、视频编解码 |
| 容器服务 | 容器编排管理 | 微服务架构 |
| 函数计算 | 事件驱动计算 | 短时任务、API 网关 |

#### 存储类服务

| 服务名称 | 功能描述 | 适用场景 |
|---------|---------|---------|
| 对象存储（OSS） | 海量非结构化数据存储 | 图片、视频、日志 |
| 块存储（云盘） | 块级存储设备 | 系统盘、数据盘 |
| 文件存储（NAS） | 文件系统存储 | 共享文件、企业应用 |
| 归档存储 | 低成本长期存储 | 数据归档、备份 |
| 备份服务 | 数据备份与恢复 | 数据备份、容灾 |

#### 数据库类服务

| 服务名称 | 功能描述 | 适用场景 |
|---------|---------|---------|
| 关系型数据库（RDS） | MySQL、PostgreSQL、SQL Server | 事务型应用 |
| NoSQL 数据库 | MongoDB、Redis、Cassandra | 缓存、内容管理 |
| 分布式数据库 | PolarDB、TDSQL、TiDB | 高并发、大数据量 |
| 数据仓库 | AnalyticDB、Redshift、BigQuery | OLAP 分析 |
| 时序数据库 | InfluxDB、Lindorm | IoT、监控数据 |

#### 网络类服务

| 服务名称 | 功能描述 | 适用场景 |
|---------|---------|---------|
| 虚拟私有云（VPC） | 隔离的虚拟网络环境 | 企业网络隔离 |
| 负载均衡（SLB/ALB） | 流量分发 | 高可用、负载分担 |
| 内容分发网络（CDN） | 全球内容加速 | 静态资源加速 |
| NAT 网关 | 网络地址转换 | 内网访问外网 |
| VPN 网关 | VPN 连接 | 远程办公、混合云 |
| 专线 | 物理专线连接 | 高速、稳定的网络连接 |

#### 安全类服务

| 服务名称 | 功能描述 | 适用场景 |
|---------|---------|---------|
| Web 应用防火墙（WAF） | 防护 Web 攻击 | 网站安全 |
| DDoS 防护 | 防护 DDoS 攻击 | 防御流量攻击 |
| 访问控制（IAM） | 身份认证与权限管理 | 用户权限控制 |
| 安全中心 | 统一安全监控 | 安全态势感知 |
| 密钥管理（KMS） | 密钥管理 | 数据加密 |
| SSL 证书 | HTTPS 证书管理 | 网站加密 |

#### 大数据与 AI 类服务

| 服务名称 | 功能描述 | 适用场景 |
|---------|---------|---------|
| 大数据计算 | MapReduce、Spark | 批量数据处理 |
| 数据湖 | 集中式数据存储 | 数据湖建设 |
| 机器学习平台 | 模型训练与部署 | AI 应用开发 |
| 图像识别 | OCR、人脸识别 | 图像分析 |
| 语音识别 | 语音转文字 | 语音处理 |
| 自然语言处理 | 文本分析、翻译 | 智能客服 |

## 二、云服务选型方法论

### 2.1 选型决策流程

```
需求分析 → 服务候选 → 评估对比 → POC 验证 → 最终决策
```

#### 第一步：需求分析
- 功能需求：需要什么功能？
- 性能需求：QPS、响应时间、并发量？
- 成本预算：总体成本是多少？
- 合规要求：数据安全、合规认证？
- 团队技能：团队熟悉哪些技术？
- 服务级别：SLA 要求是什么？

#### 第二步：服务候选
- 列出满足基本需求的服务列表
- 同一服务可能有多个产品（如数据库有多种）
- 考虑云厂商的服务差异

#### 第三步：评估对比
- 使用评估矩阵进行量化对比
- 综合技术、成本、生态等维度

#### 第四步：POC 验证
- 搭建测试环境
- 模拟真实业务场景
- 性能测试和成本验证

#### 第五步：最终决策
- 综合评估结果
- 考虑长期发展规划
- 制定迁移计划

### 2.2 选型评估矩阵

### 2.3 典型场景选型

#### 场景 1：Web 应用架构

**需求分析：**
- 访问量：日均 PV 100万
- 响应时间：< 200ms
- 可用性：99.9%
- 预算：月成本 < 1万元

**推荐架构：**
```
用户 → CDN → SLB → ECS (Web) → RDS (MySQL) → Redis (缓存)
          ↓
       OSS (静态资源)
```

**选型建议：**
| 层级 | 服务类型 | 配置建议 | 预估月成本 |
|------|---------|---------|-----------|
| CDN | 基础 CDN | 带宽 100GB | ¥500 |
| SLB | 应用负载均衡 | 共享型实例 | ¥300 |
| ECS | 2C4G × 3 | 按量付费+包年包月 | ¥6000 |
| RDS | MySQL 2C8G | 高可用版 | ¥2500 |
| Redis | 2G | 主从版 | ¥500 |
| OSS | 标准 100GB | 10TB 请求/月 | ¥200 |
| **总计** | | | **¥10000** |

#### 场景 2：微服务架构

**需求分析：**
- 服务数量：20+ 微服务
- 每日调用量：1亿+
- 扩展需求：自动伸缩
- DevOps 需求：CI/CD

**推荐架构：**
```
用户 → SLB → API Gateway → Kubernetes (微服务)
                          ↓
                        服务网格 (Istio)
                          ↓
                      消息队列 → 任务处理
                          ↓
                      Redis + RDS + MongoDB
```

**选型建议：**
| 层级 | 服务类型 | 推荐 | 理由 |
|------|---------|------|------|
| 容器编排 | ACK/TKE/EKS | 托管版 | 降低运维成本 |
| API 网关 | API Gateway | 云厂商API网关 | 集成好、功能全 |
| 服务网格 | ASM (阿里云) | 云原生网格 | 流量管理、安全 |
| 消息队列 | RocketMQ/Kafka | 托管版 | 高吞吐、可靠 |
| 监控 | Prometheus + Grafana | 云监控 | 可观测性 |

#### 场景 3：大数据分析

**需求分析：**
- 数据量：每日 1TB
- 分析延迟：< 1小时
- 成本：优先考虑性价比
- 技术栈：Hadoop 生态

**推荐架构：**
```
数据源 → 数据采集 → 数据湖 (OSS/HDFS) → 大数据计算 (EMR) → 数据分析 (QuickBI)
               ↓
         实时计算 (Flink) → 实时数据看板
```

**选型建议：**
| 层级 | 服务类型 | 推荐 | 理由 |
|------|---------|------|------|
| 数据采集 | DataHub/DTS | 云厂商数据集成 | 实时、稳定 |
| 数据湖 | OSS/HDFS | OSS + EMR | 成本低、扩展强 |
| 大数据计算 | EMR | 托管 Hadoop | 开箱即用 |
| 实时计算 | Flink | 托管 Flink | 低延迟 |
| 数据可视化 | QuickBI/DataV | 云厂商BI工具 | 简单易用 |

#### 场景 4：AI/机器学习

**需求分析：**
- 任务类型：图像识别
- 数据量：100万 张图片
- 模型训练：每周 1 次
- 推理服务：QPS 1000

**推荐架构：**
```
数据准备 → 数据标注 → 模型训练 (PAI) → 模型部署 (ECS/GPU) → 在线推理 API
                ↓
            模型管理 (模型仓库)
```

**选型建议：**
| 层级 | 服务类型 | 推荐 | 理由 |
|------|---------|------|------|
| 数据标注 | 数据标注服务 | 云厂商标注平台 | 简单高效 |
| 模型训练 | PAI/SageMaker | 云厂商AI平台 | GPU 实例丰富 |
| 模型部署 | 容器服务 + GPU | GPU 实例 | 灵活可控 |
| 在线推理 | ECS/GPU 或 PAI-EAS | 托推理服务 | 扩展性强 |

## 三、选型最佳实践

### 3.1 成本优化原则

#### 选择合适的服务等级
| 服务等级 | 适用场景 | 成本优化 |
|---------|---------|---------|
| 基础版 | 测试环境、非关键业务 | 成本最低 |
| 标准版 | 一般生产环境 | 平衡性能与成本 |
| 企业版 | 高性能、高可用要求 | 成本最高但性能最好 |

#### 选择合适的计费模式
| 计费模式 | 适用场景 | 优势 |
|---------|---------|------|
| 按量付费 | 短期使用、波动大 | 灵活、无固定成本 |
| 包年包月 | 长期稳定业务 | 折扣大（3-5折） |
| 抢占式实例 | 可中断任务 | 成本最低（1-2折） |
| 节省计划 | 长期稳定使用 | 自适应折扣 |

#### 选择合适的实例规格
- 性能基准测试
- 评估资源利用率
- 避免过度配置

### 3.2 避免供应商锁定

#### 标准化技术栈
- 使用开源技术：Kubernetes、MySQL、Redis
- 使用标准接口：RESTful API、SQL
- 避免使用云厂商专有服务

#### 多云架构设计
- 使用多云管理平台
- 选择多云兼容的服务
- 设计可迁移的架构

### 3.3 渐进式选型

#### MVP 阶段（最小可行产品）
- 优先选择成熟服务
- 快速验证业务假设
- 关注成本效率

#### 成长期阶段
- 根据实际需求优化服务
- 考虑性能和成本平衡
- 架构可扩展性

#### 成熟期阶段
- 企业级服务选型
- 高可用、容灾、安全
- 成本持续优化

## 四、选型常见误区

### 4.1 技术选型常见问题

| 误区 | 说明 | 正确做法 |
|------|------|---------|
| 过度设计 | 过早使用复杂技术 | 从简单开始，按需扩展 |
| 跟风选型 | 选择最新最热的技术 | 选择成熟稳定的技术 |
| 忽视成本 | 只看功能不考虑成本 | 综合评估 TCO |
| 单一依赖 | 只依赖一家云厂商 | 考虑多云策略 |
| 忽视运维 | 只考虑部署不考虑运维 | 评估全生命周期成本 |

### 4.2 商务选型常见问题

| 误区 | 说明 | 正确做法 |
|------|------|---------|
| 只看单价 | 忽略隐性成本 | 考虑 TCO |
| 过度谈判 | 纠结细节折扣 | 关注整体价值 |
| 忽略服务 | 只看价格不看服务 | 综合评估服务支持 |
| 缺乏规划 | 没有长期规划 | 制定 3-5 年规划 |

## 六、总结

云服务选型是架构设计的关键环节，需要综合考虑功能、性能、成本、合规等多维度因素。

**关键要点：**
1. 明确需求是选型的基础
2. 使用评估矩阵量化对比
3. POC 验证必不可少
4. 成本优化贯穿始终
5. 避免供应商锁定
6. 记录决策过程

**下一步行动：**
- 针对具体业务场景进行选型
- 建立选型评估矩阵
- 进行 POC 验证
- 持续优化和复查
