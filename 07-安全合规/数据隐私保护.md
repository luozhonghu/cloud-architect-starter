# 安全合规 - 数据隐私保护

## 一、数据隐私概述

### 1.1 数据隐私的重要性

数据隐私保护是云上业务的重要合规要求，涉及个人信息的收集、存储、使用、传输等各个环节。

**核心法律法规：**
- 《中华人民共和国个人信息保护法》（PIPL）
- 《中华人民共和国网络安全法》（CSL）
- 《中华人民共和国数据安全法》（DSL）
- GDPR（通用数据保护条例）
- HIPAA（健康保险可携带性和责任法案）

### 1.2 数据隐私保护原则

| 原则 | 说明 | 实践 |
|------|------|------|
| 最小化 | 只收集必要的数据 | 评估数据收集的必要性 |
| 目的限定 | 明确数据使用目的 | 在隐私政策中说明 |
| 透明化 | 告知用户数据处理 | 隐私政策、用户协议 |
| 用户控制 | 用户可控制数据 | 提供查询、删除接口 |
| 安全保障 | 保护数据安全 | 加密、访问控制 |

## 二、数据分类与分级

### 2.1 数据分类

| 数据类型 | 示例 | 敏感级别 |
|---------|------|---------|
| 个人身份信息 | 姓名、身份证号 | 高 |
| 个人联系方式 | 手机号、邮箱 | 高 |
| 生物识别信息 | 指纹、人脸 | 极高 |
| 金融信息 | 银行卡号、交易记录 | 极高 |
| 健康信息 | 病历、体检报告 | 高 |
| 位置信息 | GPS、IP 地址 | 中 |
| 设备信息 | 设备 ID、MAC 地址 | 低 |

### 2.2 数据分级

| 等级 | 定义 | 保护要求 |
|------|------|---------|
| 极高 | 泄露会造成重大危害 | 最高级别保护，双重加密 |
| 高 | 泛露会造成严重危害 | 加密存储传输，严格访问控制 |
| 中 | 泛露会造成一定危害 | 加密传输，一般访问控制 |
| 低 | 泛露影响较小 | 基本保护措施 |

### 2.3 数据分类分级示例

```python
class DataClassifier:
    def __init__(self):
        self.rules = {
            'id_card': {'level': '极高', 'type': '个人身份信息'},
            'phone': {'level': '高', 'type': '个人联系方式'},
            'email': {'level': '高', 'type': '个人联系方式'},
            'bank_card': {'level': '极高', 'type': '金融信息'},
            'gps': {'level': '中', 'type': '位置信息'},
            'device_id': {'level': '低', 'type': '设备信息'}
        }

    def classify(self, field_name):
        """分类数据字段"""
        return self.rules.get(field_name, {'level': '低', 'type': '一般数据'})

# 使用示例
classifier = DataClassifier()
print(classifier.classify('id_card'))    # {'level': '极高', 'type': '个人身份信息'}
print(classifier.classify('phone'))     # {'level': '高', 'type': '个人联系方式'}
print(classifier.classify('device_id')) # {'level': '低', 'type': '一般数据'}
```

#### 邮箱脱敏

```python
def mask_email(email):
    """邮箱脱敏"""
    if not email or '@' not in email:
        return email
    local, domain = email.split('@')
    if len(local) > 2:
        masked_local = local[0] + '***' + local[-1]
    else:
        masked_local = '***'
    return masked_local + '@' + domain

# 使用示例
print(mask_email('user@example.com'))  # u***r@example.com
```

#### 银行卡号脱敏

```python
def mask_bank_card(card_no):
    """银行卡号脱敏"""
    if not card_no:
        return card_no
    # 移除空格和横线
    card_no = card_no.replace(' ', '').replace('-', '')
    if len(card_no) < 6:
        return card_no
    return card_no[:4] + '******' + card_no[-4:]

# 使用示例
print(mask_bank_card('6222021234567890123'))  # 6222******90123
```

```python
from alibabacloud_kms20160120.client import Client
from alibabacloud_tea_openapi import models as open_api_models

# 创建 KMS 客户端
config = open_api_models.Config(
    access_key_id='your-access-key',
    access_key_secret='your-secret-key'

# 加密数据
def encrypt_data(plaintext, key_id):
    """加密数据"""
    response = client.encrypt(
        models.EncryptRequest(
            key_id=key_id,
            plaintext=plaintext
        )
    )
    return response.body.ciphertext_blob

# 解密数据
def decrypt_data(ciphertext_blob):
    """解密数据"""
    response = client.decrypt(
        models.DecryptRequest(
            ciphertext_blob=ciphertext_blob
        )
    )
    return response.body.plaintext.decode('utf-8')

# 使用示例
```

#### AES 加密

```python
from Crypto.Cipher import AES
from Crypto.Util.Padding import pad, unpad
import base64

def encrypt_aes(plaintext, key, iv):
    """AES 加密"""
    cipher = AES.new(key.encode('utf-8'), AES.MODE_CBC, iv.encode('utf-8'))
    padded_data = pad(plaintext.encode('utf-8'), AES.block_size)
    ciphertext = cipher.encrypt(padded_data)
    return base64.b64encode(ciphertext).decode('utf-8')

def decrypt_aes(ciphertext, key, iv):
    """AES 解密"""
    cipher = AES.new(key.encode('utf-8'), AES.MODE_CBC, iv.encode('utf-8'))
    encrypted_data = base64.b64decode(ciphertext)
    padded_data = cipher.decrypt(encrypted_data)
    plaintext = unpad(padded_data, AES.block_size)
    return plaintext.decode('utf-8')

# 使用示例
key = '01234567890123456'  # 16 字节密钥
iv = '0123456789012345'    # 16 字节 IV
plaintext = '敏感信息'
ciphertext = encrypt_aes(plaintext, key, iv)
decrypted_text = decrypt_aes(ciphertext, key, iv)

print(f'原文: {plaintext}')
print(f'密文: {ciphertext}')
print(f'解密: {decrypted_text}')
```

## 五、访问控制

### 5.1 数据访问控制

#### RBAC 权限控制

```python
class DataAccessControl:
    def __init__(self):
        self.roles = {
            'admin': ['*'],
            'data_manager': ['read', 'create', 'update', 'delete'],
            'data_viewer': ['read'],
            'data_analyst': ['read', 'export']
        }
        self.users = {
            'user1': 'admin',
            'user2': 'data_manager',
            'user3': 'data_viewer',
            'user4': 'data_analyst'
        }

    def check_permission(self, user_id, permission):
        """检查用户权限"""
        role = self.users.get(user_id)
        if not role:
            return False

        permissions = self.roles.get(role, [])
        if '*' in permissions:
            return True

        return permission in permissions

    def can_access_data(self, user_id, data_level):
        """检查用户是否可以访问指定级别的数据"""
        role = self.users.get(user_id)
        if role == 'admin':
            return True
        elif role in ['data_manager']:
            return data_level in ['极高', '高', '中', '低']
        elif role == 'data_analyst':
            return data_level in ['中', '低']
        elif role == 'data_viewer':
            return data_level in ['低']
        else:
            return False

# 使用示例
dac = DataAccessControl()

# 检查权限
print(dac.check_permission('user1', 'delete'))  # True (admin)
print(dac.check_permission('user2', 'delete'))  # True (data_manager)
print(dac.check_permission('user3', 'delete'))  # False (data_viewer)

# 检查数据访问
print(dac.can_access_data('user1', '极高'))  # True (admin)
print(dac.can_access_data('user2', '高'))    # True (data_manager)
print(dac.can_access_data('user3', '低'))     # True (data_viewer)
print(dac.can_access_data('user3', '高'))     # False (data_viewer)
```

## 八、总结

1. 遵循数据隐私保护原则
2. 对数据进行分类分级
3. 对敏感数据进行脱敏
4. 对数据进行加密保护
5. 实施严格的访问控制
