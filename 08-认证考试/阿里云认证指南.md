### 2.3 知识点详解

#### ECS 知识点

**实例类型选择**
```
通用型：Web 应用、中小型企业应用
计算优化型：高计算需求（编解码、科学计算）
内存优化型：大内存需求（数据库、缓存）
存储优化型：高 IOPS 需求（数据库）
GPU 实例：AI、渲染、计算
```

**镜像类型**
- 公共镜像：官方提供
- 自定义镜像：用户创建
- 共享镜像：他人共享
- 镜像市场：第三方应用镜像

**存储类型对比**
```
高效云盘：一般性能，成本适中
SSD 云盘：高性能，成本较高
ESSD：最高性能，成本最高
本地盘：性能最高，可靠性低
```

#### VPC 知识点

**VPC 组件**
- 专有网络（VPC）
- 交换机（VSwitch）
- 路由表
- 弹性网卡

**安全组 vs 网络ACL**
| 特性 | 安全组 | 网络ACL |
|------|--------|---------|
| 作用范围 | 实例 | 子网 |
| 状态 | 有状态 | 无状态 |
| 默认策略 | 拒绝入，允许出 | 允许所有 |

#### SLB 知识点

**SLB 类型**
- 四层（TCP/UDP）：性能高，适合非 HTTP
- 七层（HTTP/HTTPS）：功能丰富，支持会话保持

**健康检查**
- TCP 检查：检查端口连通性
- HTTP 检查：检查 HTTP 状态码

#### RDS 知识点

**实例类型**
- 基础版：单实例
- 高可用版：一主一备
- 集群版：一主多备

**备份策略**
- 自动备份：全量+增量
- 手动备份
- 按时间点恢复

### 2.6 真题示例

#### 单选题示例

**题目**：某电商企业，日访问量约 100 万 PV，需要支持高并发访问，同时成本要可控，推荐使用哪种 ECS 实例类型？

A. 计算优化型
B. 通用型
C. 内存优化型
D. GPU 实例

**答案**：B

**解析**：电商 Web 应用属于通用场景，通用型实例最合适，性价比高。

---

**题目**：需要将数据库部署在 VPC 内，只允许应用服务器访问，应如何配置安全组？

A. 开放 3306 端口给 0.0.0.0/0
B. 开放 3306 端口给应用服务器 IP
C. 开放所有端口给应用服务器 IP
D. 关闭安全组

**答案**：B

**解析**：最小权限原则，只开放必要的端口给指定的 IP。

---

#### 多选题示例

**题目**：以下哪些是 ESSD 的优势？（多选）

A. 性能高
B. 支持多副本
C. 成本低
D. 支持快照

**答案**：A、B、D

**解析**：ESSD 性能高、支持多副本、支持快照，但成本不是最低。

- 成本优化策略
- 资源规划

### 3.3 知识点详解

#### 架构设计原则

**可靠性原则**
- 冗余设计
- 故域隔离
- 快速恢复

**可扩展性原则**
- 水平扩展
- 垂直扩展
- 弹性伸缩

**安全性原则**
- 最小权限
- 纵深防御
- 零信任

**可维护性原则**
- 简化设计
- 模块化
- 文档化

#### 高可用架构

**可用性等级**
- 99%：非关键业务
- 99.9%：一般业务
- 99.95%：重要业务
- 99.99%：核心业务
- 99.999%：金融级业务

**高可用设计模式**
- 主备架构
- 集群架构
- 多可用区架构
- 多地域架构

#### 数据架构

**数据库选型决策树**
```
事务型应用？
  ├─ 是 → 关系型数据库（RDS）
  │      ├─ 数据量大？
  │      │   ├─ 是 → 分布式数据库（PolarDB）
  │      │   └─ 否 → RDS
  │      └─ 读多写少？
  │          ├─ 是 → RDS + 读写分离
  │          └─ 否 → RDS
  └─ 否 → 非关系型数据库
         ├─ 文档 → MongoDB
         ├─ 缓存 → Redis
         ├─ 列式 → HBase
         └─ 图 → Neo4j
```

4. **案例研究**：学习真实的架构案例
5. **模拟考试**：大量练习模拟题

#### 学习周期
- **基础学习**：1-2 个月
- **项目实践**：持续
- **模拟考试**：2-3 周
- **总计**：3-6 个月

### 3.5 考试技巧

#### 场景题解题思路
1. 明确业务场景和需求
2. 分析技术约束
3. 评估各种方案
4. 选择最优方案
5. 考虑成本和可维护性

#### 常见考点
- 高可用架构设计
- 容器化架构设计
- 微服务架构设计

- 安全架构设计

### 3.6 真题示例

#### 场景题示例

**题目**：某金融企业需要建设核心交易系统，要求 99.99% 的可用性，支持高并发，数据零丢失，请设计架构方案。

**答案要点：**
1. 多可用区部署，跨可用区高可用
2. 主备架构 + 自动故障切换
3. 分布式数据库 + 主从复制
4. 多级缓存提升性能
5. 完善的监控告警和灾备方案
6. 数据加密和访问控制
