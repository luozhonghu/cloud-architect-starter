# 案例实战 - 大型互联网架构案例

## 一、案例概述

### 1.1 案例背景

某大型电商平台，日活跃用户 1000 万，日均 PV 1 亿，峰值 QPS 10 万+，需要支撑双11等大促活动。

### 1.2 业务特点

| 特点 | 说明 |
|------|------|
| 高并发 | 峰值 QPS 10 万+ |
| 高可用 | 要求 99.99% 可用性 |
| 数据量大 | 日均数据量 10TB+ |
| 实时性要求 | 订单、支付需要实时处理 |
| 成本敏感 | 需要控制成本 |

### 1.3 技术挑战

1. 如何支撑高并发访问？
2. 如何保证系统高可用？
3. 如何保证数据一致性？
4. 如何优化系统性能？
5. 如何控制成本？

## 二、架构设计

### 2.1 整体架构

```
用户
  ↓
CDN（静态资源加速）
  ↓
WAF（Web应用防火墙）
  ↓
负载均衡（SLB/CLB）
  ↓
API网关（Spring Cloud Gateway）
  ↓
应用层（微服务）
  ↓
缓存层（Redis集群）
  ↓
消息队列（Kafka/RocketMQ）
  ↓
数据库层（MySQL分库分表）
  ↓
存储层（OSS/文件存储）
```

**使用示例：**
```python
# Redis 缓存使用
import redis

redis_client = redis.Redis(
    host='redis-cluster.example.com',
    port=6379,
    db=0,
    decode_responses=True
)

def get_product(product_id):
    """获取商品信息"""
    cache_key = f"product:{product_id}"
    
    # 从缓存获取
    product = redis_client.get(cache_key)
    if product:
        return json.loads(product)
    
    # 从数据库获取
    product = db.query("SELECT * FROM products WHERE id = %s", product_id)
    
    # 写入缓存
    redis_client.setex(cache_key, 3600, json.dumps(product))
    
    return product

def update_stock(product_id, quantity):
    """更新库存（使用 Lua 脚本）"""
    script = """
    local key = KEYS[1]
    local quantity = tonumber(ARGV[1])
    
    local stock = redis.call('GET', key)
    if not stock then
        return -1
    end
    
    stock = tonumber(stock)
    if stock < quantity then
        return 0
    end
    
    redis.call('DECRBY', key, quantity)
    return 1
    """
    
    result = redis_client.eval(script, 1, f"stock:{product_id}", quantity)
    return result
```

**分库分表示例：**
```python
# 分库分表路由
class ShardingRouter:
    def __init__(self):
        self.db_count = 16
        self.table_count = 8
    
    def get_db_index(self, user_id):
        """根据用户ID计算数据库索引"""
        return user_id % self.db_count
    
    def get_table_index(self, order_id):
        """根据订单ID计算表索引"""
        return (order_id // self.db_count) % self.table_count
    
    def route(self, user_id, order_id):
        """路由到具体的库和表"""
        db_index = self.get_db_index(user_id)
        table_index = self.get_table_index(order_id)
        
        db_name = f"order_db_{db_index}"
        table_name = f"order_{table_index}"
        
        return db_name, table_name

# 使用示例
router = ShardingRouter()
db_name, table_name = router.route(user_id=12345, order_id=67890)
# db_name: "order_db_9"
# table_name: "order_0"
```

## 七、实战总结

### 7.1 关键要点

1. **分层架构**：清晰分层，职责明确
2. **微服务化**：服务解耦，独立部署
3. **缓存优先**：多级缓存，提升性能
4. **消息队列**：异步处理，削峰填谷
5. **分库分表**：水平扩展，提升容量
6. **多可用区**：高可用部署，故障自动切换
7. **监控告警**：实时监控，及时告警
8. **成本优化**：按需分配，优化成本

### 7.2 最佳实践

1. **渐进式演进**：从单体到微服务，逐步演进
2. **容错设计**：熔断降级，限流保护
3. **数据一致性**：最终一致性，分布式事务
4. **安全防护**：WAF、DDoS、数据加密
5. **性能优化**：持续优化，提升性能
6. **成本控制**：合理规划，控制成本

### 7.3 经验教训

1. **不要过度设计**：根据实际需求设计架构
2. **重视监控**：监控是发现问题的重要手段
3. **做好压测**：提前压测，发现问题
4. **准备预案**：故障预案，快速响应
5. **持续学习**：技术迭代，持续学习
