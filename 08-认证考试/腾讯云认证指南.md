### 2.3 知识点详解

#### CVM 知识点

**实例类型选择**
```
通用型（S系列）：Web 应用、中小型企业应用
计算优化型（C系列）：高计算需求（编解码、科学计算）
内存优化型（M系列）：大内存需求（数据库、缓存）
存储优化型（I系列）：高 IOPS 需求（数据库）
GPU 实例（GN/GT 系列）：AI、渲染、计算
```

**计费模式对比**
| 计费模式 | 适用场景 | 优势 |
|---------|---------|------|
| 按量付费 | 短期使用、波动大 | 灵活、无固定成本 |
| 包年包月 | 长期稳定使用 | 折扣大 |
| 竞价实例 | 可中断任务 | 成本极低 |

#### VPC 知识点

**VPC 组件**
- 私有网络（VPC）
- 子网（Subnet）
- 路由表
- 弹性网卡

**安全组 vs 网络ACL**
| 特性 | 安全组 | 网络ACL |
|------|--------|---------|
| 作用范围 | 实例 | 子网 |
| 状态 | 有状态 | 无状态 |
| 默认策略 | 拒绝入，允许出 | 允许所有 |

#### CLB 知识点

**CLB 类型**
- 四层（TCP/UDP）：性能高，适合非 HTTP
- 七层（HTTP/HTTPS）：功能丰富，支持会话保持

**健康检查**
- TCP 检查：检查端口连通性
- HTTP 检查：检查 HTTP 响应码

### 2.5 真题示例

#### 单选题示例

**题目**：某电商企业，日访问量约 100 万 PV，需要支持高并发访问，同时成本要可控，推荐使用哪种 CVM 实例类型？

A. 计算优化型（C系列）
B. 通用型（S系列）
C. 内存优化型（M系列）
D. GPU 实例（GN系列）

**答案**：B

**解析**：电商 Web 应用属于通用场景，通用型实例性价比高。

---

**题目**：需要将数据库部署在 VPC 内，只允许应用服务器访问，应如何配置安全组？

A. 开放 3306 端口给 0.0.0.0/0
B. 开放 3306 端口给应用服务器 IP
C. 开放所有端口给应用服务器 IP
D. 关闭安全组

**答案**：B

**解析**：最小权限原则，只开放必要的端口给指定的 IP。

---

#### 多选题示例

**题目**：以下哪些是云硬盘（CBS）的优势？（多选）

A. 高性能
B. 支持快照
C. 成本低
D. 支持弹性扩容

**答案**：A、B、D

**解析**：云硬盘性能高、支持快照、支持弹性扩容，但成本不是最低。

## 三、TCP+ 云架构师认证

### 3.1 考试概述

**考试基本信息：**
- **考试时长**：150 分钟
- **题量**：70 道题
- **题型**：单选、多选、场景分析题
- **及格分数**：70 分（满分 100）
- **考试费用**：¥3,600
- **证书有效期**：3 年

### 3.2 考试大纲

#### 1. 架构设计基础（15%）
- 架构设计原则
- 架构模式
- 可扩展性设计
- 高可用设计

#### 2. 网络架构设计（15%）
- VPC 设计
- 跨可用区/跨地域架构
- 混合云架构
- 网络安全设计

#### 3. 应用架构设计（20%）
- 微服务架构
- 容器化架构（TKE）
- Serverless 架构（SCF）
- 无状态应用设计

#### 4. 数据架构设计（15%）
- 数据库选型
- 数据一致性
- 数据备份恢复
- 大数据架构

#### 5. 安全架构设计（15%）
- 身份认证和访问控制
- 网络安全
- 数据安全
- 安全合规

#### 6. 运维和监控（10%）
- 云监控
- 云审计
- 自动化运维
- 灾难恢复

#### 7. 成本优化（10%）
- 成本评估
- 成本优化策略
- 资源规划

### 3.3 知识点详解

#### 架构设计原则

**可靠性原则**
- 冗余设计
- 故域隔离
- 快速恢复

**可扩展性原则**
- 水平扩展
- 垂直扩展
- 弹性伸缩

**安全性原则**
- 最小权限
- 纵深防御
- 零信任

#### 高可用架构

**可用性等级**
- 99%：非关键业务
- 99.9%：一般业务
- 99.95%：重要业务
- 99.99%：核心业务
- 99.999%：金融级业务

**高可用设计模式**
- 主备架构
- 集群架构
- 多可用区架构
- 多地域架构

#### 数据架构

**数据库选型决策树**
```
事务型应用？
  ├─ 是 → 关系型数据库（TencentDB）
  │      ├─ 数据量大？
  │      │   ├─ 是 → 分布式数据库（TDSQL）
  │      │   └─ 否 → TencentDB
  │      └─ 读多写少？
  │          ├─ 是 → TencentDB + 读写分离
  │          └─ 否 → TencentDB
  └─ 否 → 非关系型数据库
         ├─ 文档 → MongoDB
         ├─ 缓存 → Redis
         ├─ 列式 → HBase
         └─ 图 → Neo4j
```

### 3.4 备考建议

#### 学习路径
1. **TCP 认证**：先通过 TCP 认证打基础
2. **项目经验**：至少 3 年架构设计经验
3. **深入学习**：深入学习每个技术领域
4. **案例研究**：学习真实的架构案例
5. **模拟考试**：大量练习模拟题

#### 学习周期
- **基础学习**：1-2 个月
- **项目实践**：持续
- **模拟考试**：2-3 周
- **总计**：3-6 个月

### 3.5 真题示例

#### 场景题示例

**题目**：某金融企业需要建设核心交易系统，要求 99.99% 的可用性，支持高并发，数据零丢失，请设计架构方案。

**答案要点：**
1. 多可用区部署，跨可用区高可用
2. 主备架构 + 自动故障切换
3. 分布式数据库 + 主从复制
4. 多级缓存提升性能
5. 完善的监控告警和灾备方案
6. 数据加密和访问控制
