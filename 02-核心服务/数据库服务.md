# 核心服务 - 数据库服务（RDS/NoSQL）

## 一、数据库服务概述

### 1.1 什么是云数据库服务？

云数据库服务是云厂商托管的数据库服务，提供自动备份、故障切换、性能监控等功能，用户无需关注数据库的底层维护。

**核心价值：**
- **高可用**：主备架构，故障自动切换
- **自动备份**：定时备份，时间点恢复
- **弹性扩展**：一键扩容，无需停机
- **安全性**：VPC 隔离、SSL 加密、访问控制
- **运维简化**：自动维护升级，监控告警

### 1.2 数据库服务类型

| 类型 | 典型产品 | 适用场景 |
|------|---------|---------|
| 关系型数据库 | RDS MySQL/PostgreSQL/SQL Server | 事务型应用、关系数据 |
| NoSQL 数据库 | MongoDB、Redis、Cassandra | 非结构化数据、缓存 |
| 分布式数据库 | PolarDB、TDSQL、TiDB | 高并发、大数据量 |
| 数据仓库 | AnalyticDB、Redshift、BigQuery | OLAP 分析 |
| 时序数据库 | Lindorm、InfluxDB | IoT、监控数据 |

### 1.3 各厂商数据库服务对比

| 服务名称 | 云厂商 | 特点 |
|---------|-------|------|
| RDS | 阿里云 | 支持多种数据库引擎 |
| TencentDB | 腾讯云 | 性价比高，性能稳定 |
| GaussDB | 华为云 | 自研能力强，分布式数据库领先 |
| RDS | AWS | 全球领先，服务丰富 |
| SQL Database | Azure | 与微软生态集成好 |

## 二、关系型数据库（RDS）

### 2.1 支持的数据库引擎

#### MySQL
- 最流行的开源关系型数据库
- 社区活跃，文档丰富
- 适合 Web 应用、电商、社交等

#### PostgreSQL
- 高级开源数据库
- 支持复杂查询、GIS 数据
- 适合企业应用、数据分析

#### SQL Server
- 微软商业数据库
- Windows 生态集成好
- 适合微软技术栈应用

### 2.2 实例类型选择

#### 基础版
- 单实例，无备机
- 成本最低
- 适合测试、开发环境
- ⚠️ 无高可用保障

#### 高可用版
- 主备架构，一主一备
- 故障自动切换
- 适合生产环境
- 推荐选择

#### 集群版
- 一主多备
- 读写分离
- 高并发场景
- 成本较高

### 2.3 规格选择

#### 内存型（r 系列）
- 大内存配置
- 适合缓存、内存数据库
- 高并发读写

#### 通用型（g 系列）
- 平衡计算和内存
- 适合大多数应用
- 性价比高

#### 存储优化型（i 系列）
- 高 IOPS
- 适合数据库、数据仓库
- 存储密集型应用

#### 规格选择建议

| 业务场景 | 推荐规格 | 实例数 |
|---------|---------|--------|
| 小型网站 | 1C2G 或 2C4G | 1-2 |
| 中型应用 | 2C8G 或 4C16G | 2-4 |
| 大型应用 | 8C32G 或 16C64G | 4+ |
| 高并发 | 16C64G+ 集群版 | 8+ |

### 2.4 存储类型

#### 云盘存储
- SSD 云盘：高性能
- ESSD（增强型SSD）：最高性能
- 成本：ESSD > SSD > 高效云盘

#### 本地盘
- 性能最强
- 价格较低
- 可靠性依赖实例

## 三、NoSQL 数据库

### 3.1 Redis（缓存）

#### 适用场景
- 缓存
- 会话存储
- 排行榜
- 计数器
- 消息队列（轻量级）

#### 实例类型
- **单副本**：单实例，低成本
- **集群版**：分布式，高可用
- **读写分离版**：主从架构

#### 规格选择

| 业务规模 | 推荐规格 | 内存 |
|---------|---------|------|
| 小型 | 1GB | 缓存基础数据 |
| 中型 | 4-8GB | 缓存热点数据 |
| 大型 | 16-64GB | 大规模缓存 |
| 超大型 | 128GB+ | 集群版 |

#### 使用示例
```python
import redis

# 连接 Redis
r = redis.Redis(host='r-xxxxx.redis.rds.aliyuncs.com', port=6379, password='xxx')

# 设置键值
r.set('key', 'value')

# 获取值
value = r.get('key')

# 设置过期时间（秒）
r.setex('session:123', 'user_data', 3600)

# 批量操作
pipe = r.pipeline()
pipe.set('key1', 'value1')
pipe.set('key2', 'value2')
pipe.execute()
```

### 4.2 典型产品

| 产品 | 云厂商 | 特点 |

| TiDB | 独立/云厂商 | 开源分布式数据库 |
| GaussDB | 华为云 | 企业级分布式数据库 |

### 4.3 架构特点

#### 存储计算分离
- 存储层：分布式存储
- 计算层：可独立扩展

#### 分布式事务
- 两阶段提交（2PC）
- 最终一致性

#### 自动分片
- 数据自动分布
- 负载均衡

## 五、数据库连接与访问

### 5.1 连接方式

#### 内网连接
- 同一 VPC 内访问
- 安全、稳定
- 推荐生产环境使用

#### 公网连接
- 通过公网 IP 访问
- ⚠️ 安全风险，不推荐
- 适合临时访问

#### 专线连接
- 通过专线连接
- 稳定、高速
- 适合混合云

### 5.2 连接池配置

```java
// HikariCP 连接池示例
HikariConfig config = new HikariConfig();
config.setJdbcUrl("jdbc:mysql://rm-xxxxx.mysql.rds.aliyuncs.com:3306/mydb");
config.setUsername("username");
config.setPassword("password");
config.setMaximumPoolSize(20); // 最大连接数
config.setMinimumIdle(5); // 最小空闲连接
config.setConnectionTimeout(30000); // 连接超时
config.setIdleTimeout(600000); // 空闲超时

HikariDataSource dataSource = new HikariDataSource(config);
```

### 5.3 读写分离

## 六、数据库安全

### 6.1 网络安全
- **VPC 隔离**：数据库部署在 VPC 内
- **白名单设置**：只允许特定 IP 访问
- **SSL 加密**：数据传输加密

### 6.2 访问控制
- **账号权限**：最小权限原则
- **密码策略**：强密码、定期更换
- **审计日志**：记录访问操作

### 6.3 数据加密
- **传输加密**：SSL/TLS
- **存储加密**：TDE（透明数据加密）
- **备份加密**：加密备份数据

### 7.1 备份策略

- 定期全量备份
- 增量日志备份

### 8.2 查询优化

## 九、高可用与灾备

### 9.1 主从复制
- 异步复制：性能好，可能数据丢失
- 半同步复制：折中方案
- 同步复制：数据一致，性能差

### 9.2 读写分离
- 读操作分发到从库
- 写操作在主库执行
- 提升读性能

### 9.3 故障切换
- 自动检测故障
- 自动切换到备库
- 应用透明

### 9.4 异地灾备
- 跨区域数据同步
- 灾难恢复演练
- RTO/RPO 指标
