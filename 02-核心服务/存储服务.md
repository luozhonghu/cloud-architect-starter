# 核心服务 - 存储服务（OSS/S3）

## 一、存储服务概述

### 1.1 什么是对象存储？

对象存储（Object Storage Service，简称 OSS/S3）是一种海量、安全、低成本、高可靠的云存储服务，用于存储图片、音视频、日志等非结构化数据。

**核心特性：**
- **海量存储**：支持 PB 级数据存储
- **高可靠**：99.9999999999%（12个9）的数据可靠性
- **高可用**：99.99% 的服务可用性
- **低成本**：按使用量付费，无需预置
- **弹性扩展**：自动扩展，无需手动扩容

### 1.2 各厂商存储服务对比

| 服务名称 | 云厂商 | 特点 |
|---------|-------|------|
| OSS | 阿里云 | 国内最成熟，功能齐全 |
| COS | 腾讯云 | 与微信小程序集成好 |
| OBS | 华为云 | 安全性能强 |
| S3 | AWS | 行业标准，全球覆盖 |
| Blob Storage | Azure | Windows 生态好 |

### 1.3 对象存储的核心概念

#### 存储桶（Bucket）
存储对象的容器，类似文件夹。

#### 对象（Object）
存储的基本单位，包含数据和元数据。

#### 密钥（Key）
对象的唯一标识符。

#### 区域（Region）
数据中心所在的地理位置。

#### 访问域名（Endpoint）
访问存储服务的网络地址。

## 二、存储类型选择

### 2.1 存储类型对比

| 存储类型 | 访问频率 | 访问延迟 | 最低存储时长 | 成本（元/GB/月） | 适用场景 |
|---------|---------|---------|-------------|-----------------|---------|
| 标准存储 | 频繁 | 毫秒级 | 无 | 0.12 | 热数据、频繁访问 |
| 低频存储 | 不频繁 | 秒级 | 30天 | 0.08 | 备份数据、日志 |
| 归档存储 | 很少访问 | 分钟级 | 90天 | 0.033 | 合规归档、长期保存 |
| 冷归档存储 | 极少访问 | 数小时 | 180天 | 0.015 | 法规归档、数据归档 |

### 2.2 存储类型选择建议

| 业务场景 | 推荐存储类型 | 理由 |
|---------|-------------|------|
| 网站图片/视频 | 标准存储 | 访问频繁，需要快速响应 |
| 用户上传文件 | 标准存储 | 访问频繁 |
| 日志文件 | 低频存储 | 保留时间长，访问较少 |
| 备份数据 | 低频/归档存储 | 偶尔恢复访问 |
| 合规数据 | 归档/冷归档 | 法规要求长期保存 |
| 大数据原始数据 | 归档存储 | 只在需要时访问 |

### 2.3 存储生命周期管理

#### 自动转换策略
```
对象创建 → 30天后 → 低频存储 → 90天后 → 归档存储 → 180天后 → 冷归档存储
```

#### 配置示例（阿里云 OSS）
```json
{
  "rules": [
    {
      "id": "rule1",
      "status": "Enabled",
      "prefix": "logs/",
      "expiration": {
        "days": 365
      },
      "transitions": [
        {
          "days": 30,
          "storageClass": "IA"
        },
        {
          "days": 90,
          "storageClass": "Archive"
        }
      ]
    }
  ]
}
```

## 三、存储桶配置

### 3.1 存储桶命名规则
- 长度：3-63 个字符
- 只能包含小写字母、数字、连字符（-）
- 必须以字母或数字开头和结尾
- 全局唯一（在整个云厂商内）

**正确示例：**
- `my-bucket`
- `app-images-2024`
- `prod-backup-files`

**错误示例：**
- `My-Bucket`（包含大写）
- `-my-bucket`（以连字符开头）
- `my_bucket`（包含下划线）

### 3.2 存储桶访问权限

#### 公共读
- 任何人都可以读取对象
- 适合公开的静态资源
- ⚠️ 安全风险，谨慎使用

#### 私有
- 只有授权用户可以访问
- 推荐的默认设置
- 通过签名 URL 或临时凭证分享

#### 公共读写
- 任何人都可以读写
- ⚠️ 极高风险，不推荐

### 3.3 跨域配置（CORS）

允许跨域访问存储资源。

```json
{
  "CORSRule": [
    {
      "AllowedOrigin": ["https://example.com"],
      "AllowedMethod": ["GET", "POST", "PUT"],
      "AllowedHeader": ["*"],
      "ExposeHeader": ["ETag"],
      "MaxAgeSeconds": 3600
    }
  ]
}
```

### 3.4 静态网站托管

将存储桶配置为静态网站。

**特点：**
- 支持自定义域名
- 支持 HTTPS
- 支持默认首页、错误页面
- 适合静态网站托管

**适用场景：**
- 个人博客
- 企业官网
- 单页应用（SPA）
- 文档站点

## 四、数据安全

### 4.1 访问控制（IAM）

#### Bucket Policy
基于资源的访问控制。

```json
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Principal": {
        "AWS": "arn:aws:iam::123456789012:user/username"
      },
      "Action": [
        "s3:GetObject",
        "s3:PutObject"
      ],
      "Resource": [
        "arn:aws:s3:::my-bucket/*"
      ]
    }
  ]
}
```

#### 用户策略
基于身份的访问控制。

```json
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Action": [
        "s3:ListBucket",
        "s3:GetObject"
      ],
      "Resource": [
        "arn:aws:s3:::my-bucket",
        "arn:aws:s3:::my-bucket/*"
      ]
    }
  ]
}
```

### 4.2 数据加密

#### 服务端加密（SSE）
- **SSE-S3**：使用云厂商管理的密钥
- **SSE-KMS**：使用 KMS 管理的密钥（推荐）
- **SSE-C**：使用用户提供的密钥

#### 客户端加密
- 客户端加密后再上传
- 客户端解密数据
- 密钥由用户管理

### 4.3 版本控制

启用版本控制后，每次修改都会保留历史版本。

**优势：**
- 误删除可恢复
- 覆盖可回滚
- 提供审计能力

**成本影响：**
- 存储成本增加
- 需要定期清理历史版本

### 4.4 合规保留（WORM）

启用合规保留后，对象在保留期内无法删除或覆盖。

**适用场景：**
- 法规要求的数据保存
- 审计数据
- 重要业务数据

### 4.5 防盗链设置

设置 Referer 白名单，防止第三方盗链。

```json
{
  "RefererConfiguration": {
    "AllowEmptyReferer": false,
    "RefererList": [
      "https://example.com",
      "https://www.example.com"
    ]
  }
}
```

## 五、访问方式

### 5.1 控制台访问
- 图形化界面操作
- 适合日常管理
- 批量操作便捷

### 5.2 SDK 访问
- 提供多语言 SDK
- 适合应用集成
- 自动处理签名和重试

#### Python SDK 示例（阿里云 OSS）
```python
import oss2

# 配置认证信息
auth = oss2.Auth('AccessKeyId', 'AccessKeySecret')
bucket = oss2.Bucket(auth, 'https://oss-cn-hangzhou.aliyuncs.com', 'my-bucket')

# 上传文件
bucket.put_object('hello.txt', 'Hello OSS!')

# 下载文件
content = bucket.get_object('hello.txt').read()
print(content)

# 列出文件
for obj in oss2.ObjectIterator(bucket):
    print(obj.key)
```

### 5.3 REST API 访问
- 标准化的 HTTP API
- 适合跨平台集成
- 需要手动处理签名

### 5.4 命令行工具（CLI）

```bash
# 上传文件
aws s3 cp file.txt s3://my-bucket/

# 下载文件
aws s3 cp s3://my-bucket/file.txt .

# 同步目录
aws s3 sync ./local-dir s3://my-bucket/remote-dir

# 列出文件
aws s3 ls s3://my-bucket/
```

## 六、性能优化

### 6.1 上传优化

#### 分片上传
- 大文件（>100MB）自动分片
- 并行上传
- 断点续传
- 进度跟踪

#### 加速上传
- 使用 CDN 加速
- 选择就近的区域
- 多线程上传

### 6.2 下载优化

#### CDN 加速
- 静态资源通过 CDN 分发
- 全球节点覆盖
- 智能路由

#### 预签名 URL
- 临时授权访问
- 适合分享私密文件
- 支持设置过期时间

### 6.3 访问优化

#### 读写分离
- 读取操作访问 CDN
- 写入操作直连存储服务

#### 镜像回源
- CDN 回源拉取
- 降低回源流量

## 七、成本优化

### 7.1 成本组成

```
总成本 = 存储费用 + 流量费用 + 请求费用 + 其他费用
```

### 7.2 成本优化策略

#### 存储成本优化
- 使用生命周期管理自动转换存储类型
- 定期清理无用数据
- 压缩文件后再上传

#### 流量成本优化
- 使用 CDN 降低流量费用
- 开启流量包优惠
- 选择就近的区域

#### 请求成本优化
- 批量操作减少请求次数
- 使用预签名 URL 减少鉴权请求
- 合理设置缓存策略

## 八、最佳实践

### 8.1 存储桶规划
- 按业务或环境分桶（prod、test、dev）

- 使用临时凭证代替长期密钥

### 8.3 监控告警
- 存储容量监控
- 请求量监控
- 错误率监控
- 成本监控

### 8.4 灾备策略
- 跨区域复制（CRR）

## 十、与其他存储服务的对比

| 特性 | 对象存储 | 块存储（EBS/云盘） | 文件存储（NAS） |
|------|---------|------------------|---------------|
| 数据类型 | 非结构化数据 | 块级数据 | 文件系统数据 |
| 访问方式 | HTTP/HTTPS API | 挂载到实例 | NFS/CIFS 挂载 |
| 扩展性 | 无限扩展 | 容量有限 | 容量有限 |
| 延迟 | 毫秒级 | 毫秒级 | 毫秒级 |
| 适用场景 | 图片、视频、日志 | 系统盘、数据盘 | 共享文件、企业应用 |

## 十一、总结

对象存储是云存储的基础服务，适合存储海量非结构化数据。合理选择存储类型、配置访问控制、优化成本，可以有效支撑业务需求。

**关键要点：**
1. 根据访问频率选择存储类型
2. 访问控制是安全的基础
3. 生命周期管理自动优化成本
4. CDN 加速提升访问性能
5. 版本控制防止误删
6. 跨区域复制实现灾备
