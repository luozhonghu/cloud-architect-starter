# 成本优化 - 资源选型策略

## 一、资源选型概述

### 1.1 选型原则

资源选型是成本优化的第一步，选择合适的资源可以实现"花最少的钱，满足业务需求"。

#### 核心原则

| 原则 | 说明 | 实践 |
|------|------|------|
| 适用性 | 满足业务需求 | 功能、性能、可靠性 |
| 经济性 | 成本最优 | 性价比最高 |
| 可扩展性 | 支持业务增长 | 水平扩展、垂直扩展 |
| 可维护性 | 易于维护 | 标准化、文档化 |
| 安全性 | 满足安全要求 | 安全防护、合规 |

#### 选型决策流程

```
需求分析 → 候选方案 → 评估对比 → POC验证 → 最终决策
```

### 1.2 选型评估维度

#### 评估维度

| 维度 | 权重 | 评估指标 |
|------|------|---------|
| 功能完整性 | 25% | 是否满足功能需求 |
| 性能指标 | 20% | QPS、响应时间、并发能力 |
| 成本 | 20% | TCO、月成本、边际成本 |
| 稳定性/SLA | 15% | 可用性、故障恢复能力 |
| 文档与社区 | 10% | 文档质量、社区活跃度 |
| 团队熟悉度 | 10% | 学习成本、维护成本 |

### 2.2 计费模式选择

#### 计费模式对比

| 计费模式 | 适用场景 | 优势 | 劣势 |
|---------|---------|------|------|
| 按量付费 | 短期使用、波动大 | 灵活、无固定成本 | 单价高 |
| 包年包月 | 长期稳定使用 | 折扣大、成本确定 | 不够灵活 |
| 抢占式实例 | 可中断任务 | 成本极低 | 可能被中断 |

#### 决策树

```
是否长期使用（>1年）？
  ├─ 是 → 包年包月
  └─ 否
     是否可中断？
       ├─ 是 → 抢占式实例
       └─ 否 → 按量付费
```

#### 成本对比示例

**场景：** 需要 4 台 2C4G 实例，使用 1 年

| 计费模式 | 单价 | 月成本 | 年成本 | 节省 |
|---------|------|--------|--------|------|
| 按量付费 | ¥0.39/小时 | ¥1,123 | ¥13,476 | - |
| 包年包月 | ¥200/月 | ¥800 | ¥9,600 | 29% |
| 抢占式实例 | ¥0.05/小时 | ¥144 | ¥1,728 | 87% |

### 2.3 实例规格优化

#### 评估资源利用率

**关键指标：**
- CPU 使用率
- 内存使用率
- 磁盘 IOPS
- 网络带宽

**优化策略：**

```
CPU 使用率 > 80% → 升级规格
CPU 使用率 < 30% → 降级规格
内存使用率 > 85% → 升级规格
内存使用率 < 30% → 降级规格
```

#### 自动伸缩策略

**扩容条件：**
- CPU 使用率 > 70%，持续 5 分钟
- 内存使用率 > 80%，持续 5 分钟

**缩容条件：**
- CPU 使用率 < 30%，持续 15 分钟
- 内存使用率 < 30%，持续 15 分钟

**配置示例：**
```yaml
auto_scaling:
  min_instances: 2
  max_instances: 10
  scale_up:
    metric: cpu_usage
    threshold: 70
    duration: 5m
    scale_by: 2
  scale_down:
    metric: cpu_usage
    threshold: 30
    duration: 15m
    scale_by: 1
```

#### 按数据类型选择

| 数据类型 | 推荐存储 | 理由 |
|---------|---------|------|
| 图片/视频 | OSS 标准存储 | 访问频繁 |
| 日志文件 | OSS 低频存储 | 访问较少 |
| 数据备份 | OSS 归档存储 | 长期保存 |
| 系统盘 | SSD 云盘 | 性能要求高 |
| 数据库盘 | ESSD | 高 IOPS |

### 3.2 存储容量规划

#### 容量计算公式

```
总容量 = (数据量 × 备份倍数) + (增长空间 × 保留期) + 预留空间
```

#### 示例：电商网站存储规划

**现状：**
- 当前数据量：500GB
- 日增长率：10GB
- 备份倍数：3
- 保留期：90 天
- 预留空间：20%

```
总容量 = (500 × 3) + (10 × 90) × 1.2
       = 1,500 + 900 × 1.2
       = 1,500 + 1,080
       = 2,580GB
```

**成本：**
- 500GB 标准存储：500 × 0.12 = ¥60/月
- 2,000GB 低频存储：2,000 × 0.08 = ¥160/月
- 80GB 归档存储：80 × 0.033 = ¥2.64/月
- **总计：¥222.64/月**

### 3.3 存储生命周期管理

#### 自动转换策略

**策略：**
- 数据创建后 30 天：标准存储
- 30-90 天：低频存储
- 90 天后：归档存储
- 1 年后：冷归档存储

**配置示例（阿里云 OSS）：**
```json
{
  "rules": [
    {
      "id": "lifecycle_rule",
      "status": "Enabled",
      "prefix": "data/",
      "expiration": {
        "days": 365
      },
      "transitions": [
        {
          "days": 30,
          "storageClass": "IA"
        },
        {
          "days": 90,
          "storageClass": "Archive"
        },
        {
          "days": 365,
          "storageClass": "ColdArchive"
        }
      ]
    }
  ]
}
```

**成本优化效果：**

| 时间 | 存储类型 | 单价 | 月成本 |
|------|---------|------|--------|
| 1-30天 | 标准存储 | ¥0.12 | ¥6 |
| 31-90天 | 低频存储 | ¥0.08 | ¥4.8 |
| 91-365天 | 归档存储 | ¥0.033 | ¥8.9 |
| **平均** | | | **¥19.7** |

**不使用生命周期：**
- 全部标准存储：1,000GB × 0.12 = ¥120/月

**节省：** (120 - 19.7) / 120 = 83.6%

| 大数据分析 | HBase | 海量数据存储 |

### 4.2 数据库规格选择

#### 规格选择原则

**核心指标：**
- QPS（每秒查询数）
- TPS（每秒事务数）
- 数据量
- 并发连接数

**规格选择建议：**

| QPS | 推荐规格 | 月成本 |
|-----|---------|--------|
| < 1,000 | 1C2G | ¥200 |
| 1,000-5,000 | 2C4G | ¥400 |
| 5,000-10,000 | 2C8G | ¥600 |
| 10,000-50,000 | 4C16G | ¥1,200 |
| > 50,000 | 8C32G+ | ¥2,400+ |

#### 读写分离选型

**适用场景：**
- 读多写少
- 读 QPS > 5,000
- 需要提升读性能

**配置建议：**
- 1 主 2 从
- 读请求分配到从库
- 写请求在主库

**成本示例：**
- 主库（4C16G）：¥1,200/月
- 从库 1（4C16G）：¥1,200/月
- 从库 2（4C16G）：¥1,200/月
- **总计：¥3,600/月**

**单主库（16C64G）：¥4,800/月**

**节省：** (4,800 - 3,600) / 4,800 = 25%

| 会话保持 | 七层负载均衡 |

### 5.2 CDN 选型

#### 按内容类型选择

| 内容类型 | 推荐配置 | 理由 |
|---------|---------|------|
| 静态资源（图片/视频） | CDN 缓存 1 年 | 访问频繁，缓存时间长 |
| API 接口 | 不缓存 | 动态内容 |
| 前端资源 | CDN 缓存 1 个月 | 更新频率中等 |

#### 按流量选择计费方式

| 流量特点 | 计费方式 | 理由 |
|---------|---------|------|
| 流量波动大 | 按流量计费 | 成本可控 |
| 流量稳定 | 按带宽计费 | 成本确定 |
| 大流量活动 | 流量包 | 成本更低 |

#### 成本对比示例

**场景：** 月流量 10TB

| 计费方式 | 单价 | 月成本 |
|---------|------|--------|
| 按流量 | ¥0.5/GB | ¥5,000 |
| 流量包 10TB | ¥0.4/GB | ¥4,000 |
| 流量包 100TB | ¥0.3/GB | ¥3,000 |

**节省：** 流量包比按流量节省 20-40%

### 5.3 带宽选型

#### 按业务类型选择

| 业务类型 | 推荐带宽 | 计费方式 |
|---------|---------|---------|
| 下载业务 | 高带宽 | 按带宽计费 |
| Web 应用 | 中等带宽 | 按流量计费 |
| 视频直播 | 高带宽 | 按带宽计费 |

#### 带宽估算

**公式：**
```
带宽峰值 = QPS × 平均响应大小 × 8 / 1,000,000
```

**示例：**
- QPS：1,000
- 平均响应大小：100KB

```
带宽峰值 = 1,000 × 100 × 1,024 × 8 / 1,000,000
         = 819 Mbps
```

**推荐：** 1Gbps 带宽

## 八、总结

资源选型是成本优化的基础，合理选择资源可以实现"花最少的钱，满足业务需求"。

**关键要点：**
1. 资源选型要遵循适用性、经济性、可扩展性原则
2. 根据业务场景选择合适的实例类型
3. 选择合适的计费模式可以大幅降低成本
4. 存储生命周期管理是成本优化的关键
5. 数据库选型要考虑数据模型和使用场景
6. CDN 可以同时降低成本和提升性能
7. 定期评估和优化资源配置
