# 核心服务 - 计算服务（ECS/CVM/EC2）

## 一、计算服务概述

### 1.1 什么是计算服务？

计算服务（Elastic Compute Service，简称 ECS）是云厂商提供的最基础、最核心的 IaaS 服务，提供可弹性伸缩的云端计算能力。

**核心特性：**
- **弹性伸缩**：按需创建/释放实例
- **高可用性**：多可用区部署，故障自动恢复
- **灵活配置**：多种实例规格、操作系统、网络配置
- **按需付费**：按实际使用量计费

### 1.2 各厂商计算服务对比

| 服务名称 | 云厂商 | 特点 |
|---------|-------|------|
| ECS | 阿里云 | 实例规格最全，性能稳定 |
| CVM | 腾讯云 | 性价比高，适合游戏等场景 |
| EC2 | AWS | 全球领先，规格丰富 |
| ECS | 华为云 | 支持鲲鹏、昇腾等国产芯片 |
| Virtual Machine | Azure | Windows 支持最好 |

### 1.3 计算服务的关键概念

#### 实例（Instance）
云服务器的基本单位，一台实例相当于一台虚拟服务器。

#### 镜像（Image）
实例的模板，包含操作系统和预装软件。

#### 磁盘（Disk）
实例的块存储设备，包括系统盘和数据盘。

#### 安全组（Security Group）
虚拟防火墙，控制实例的入站和出站流量。

#### 密钥对（Key Pair）
用于 SSH 登录的密钥认证方式。

### 2.2 按计费模式分类

#### 按量付费
按实际使用时长计费，灵活但单价较高。

**特点：**
- 适合短期使用
- 不需要预付费用
- 随时释放实例
- 成本相对较高

**适用场景：**
- 临时测试
- 突发业务高峰
- 开发验证

#### 包年包月
预先付费购买，长期使用成本低。

**特点：**
- 需要预付费
- 长期使用成本优惠大
- 适合稳定业务
- 不可随意释放

**适用场景：**
- 生产环境
- 长期运行的业务
- 成本敏感的业务

#### 抢占式实例
低价获取闲置计算资源，可被系统回收。

**特点：**
- 成本极低（1-2 折）
- 可被系统回收
- 最多运行 24 小时
- 不适合关键业务

**适用场景：**
- 批量处理任务
- CI/CD 构建任务
- 数据处理
- 不紧急的计算任务

#### 节省计划
承诺一定的使用量，获取价格折扣。

**特点：**
- 灵活性高
- 折扣幅度大
- 可跨实例使用
- 适合长期稳定使用

**适用场景：**
- 长期稳定业务
- 大量实例使用
- 成本优化

## 三、镜像选择

### 3.1 镜像类型

#### 公共镜像
云厂商提供的标准镜像，包含常用操作系统。

**常见系统：**
- Linux：CentOS、Ubuntu、Debian、SUSE
- Windows：Windows Server 2012/2016/2019/2022

#### 自定义镜像
用户基于实例创建的镜像。

**用途：**
- 备份系统环境
- 批量部署相同环境
- 制作标准化的服务镜像

#### 共享镜像
其他用户共享的镜像。

**用途：**
- 获取预配置的环境
- 学习和测试

#### 镜像市场
第三方提供的应用镜像。

**典型应用：**
- LAMP/LNMP 环境
- Docker 环境
- 应用中间件
- 开发环境

### 3.2 镜像选择建议

| 业务场景 | 推荐镜像 | 理由 |
|---------|---------|------|
| Web 服务器 | CentOS 7/8 | 稳定、文档丰富 |
| 开发环境 | Ubuntu 20.04/22.04 | 软件包丰富 |
| Windows 应用 | Windows Server 2019 | 兼容性好 |
| 容器部署 | 自定义镜像（预装Docker） | 快速部署 |
| 特定应用 | 镜像市场 | 开箱即用 |

## 四、存储配置

### 4.1 系统盘

#### 云盘类型
| 类型 | 性能 | 成本 | 适用场景 |
|------|------|------|---------|
| 高效云盘 | 中等 | 中等 | 一般应用 |
| SSD 云盘 | 高 | 较高 | 数据库、高性能应用 |
| ESSD（增强型SSD） | 极高 | 高 | 核心业务、数据库 |
| 本地盘 | 最高 | 中等 | 临时数据、缓存 |

#### 容量建议
- **最小**：40GB（基础系统）
- **推荐**：40-100GB（根据应用需求）
- **注意**：系统盘容量不宜过小，影响系统更新和日志

### 4.2 数据盘

#### 选择策略
- **热数据**：使用 SSD/ESSD
- **冷数据**：使用高效云盘
- **备份**：使用归档存储
- **共享存储**：使用文件存储（NAS）

#### 容量规划
```python
# 数据盘容量估算公式
总容量 = (业务数据量 × 备份倍数) + (增长空间 × 保留期) + 预留空间
```

## 五、网络配置

### 5.1 网络类型

#### 经典网络
- 早期网络模式
- 实例之间可以直接访问
- 安全性较低
- 逐步淘汰

#### 专有网络（VPC）
- **推荐**
- 隔离的虚拟网络环境
- 安全性高
- 灵活配置

### 5.2 带宽配置

#### 按使用流量计费
- 适合访问量波动大的业务
- 下载量大的场景（视频、文件下载）

#### 按固定带宽计费
- 适合访问量稳定的业务
- 对网络性能要求高的场景
- 访问量可预测的业务

### 5.3 负载均衡
- 四层负载均衡（TCP/UDP）
- 七层负载均衡（HTTP/HTTPS）
- 会话保持
- 健康检查

## 六、安全配置

### 6.1 安全组规则

#### 入站规则示例
```
协议    端口范围    授权对象         描述
SSH     22         0.0.0.0/0       允许SSH登录（可限制IP）

HTTPS   443        0.0.0.0/0       允许HTTPS访问
MySQL   3306       10.0.0.0/8      仅允许VPC内访问
```

#### 出站规则示例
```
协议    端口范围    授权对象         描述
All     All        0.0.0.0/0       允许所有出站（可限制）
```

### 6.2 密钥对认证
- 比密码更安全
- 禁用密码登录
- 定期轮换密钥

### 6.3 其他安全措施
- 定期更新系统补丁
- 安装安全软件
- 启用云监控告警
- 定期备份重要数据

### 7.3 成本优化建议

#### 短期使用
- 选择按量付费
- 释放闲置实例
- 使用抢占式实例（如果可接受中断）

#### 长期使用
- 选择包年包月
- 使用节省计划
- 选择合适规格，避免过度配置

#### 带宽优化
- 使用 CDN 分发静态资源
- 压缩传输内容
- 选择合适的计费方式

## 八、最佳实践

### 8.1 创建实例

```bash
# 使用 CLI 创建实例（阿里云示例）
aliyun ecs CreateInstance \
  --RegionId cn-hangzhou \
  --ImageId aliyun_2_1903_x64_20G_alibase_20200324.vhd \
  --InstanceType ecs.g6.large \
  --SecurityGroupId sg-xxxxxxxx \
  --VSwitchId vsw-xxxxxxxx \
  --InternetChargeType PayByTraffic \
  --InternetMaxBandwidthOut 5
```

### 8.2 扩容策略
- 水平扩展：增加实例数量
- 垂直扩展：升级实例规格
- 自动伸缩：基于指标自动扩展

### 8.3 备份策略
- 定期创建镜像
- 使用快照备份数据
- 异地备份

### 8.4 监控告警
- CPU 使用率 > 80%
- 内存使用率 > 85%
- 磁盘使用率 > 90%
- 网络流量异常

### 8.5 生命周期管理
```
创建 → 配置 → 部署应用 → 监控 → 扩容/缩容 → 备份 → 释放
```

## 九、常见问题

### 9.1 实例无法启动
- 检查资源配额
- 检查磁盘容量
- 检查安全组规则

### 9.2 性能问题
- 升级实例规格
- 优化应用
- 检查带宽瓶颈
- 使用负载均衡

### 9.3 成本过高
- 评估实例规格是否合适
- 使用包年包月计费
- 释放闲置实例
- 使用自动伸缩
