# 认证考试 - 真题解析与备考指南

## 一、备考通用指南

### 1.1 备考策略

#### 学习路径
```
基础认知 → 核心服务学习 → 实践操作 → 模拟考试 → 查漏补缺 → 正式考试
```

#### 时间规划
| 阶段 | 时间 | 任务 | 完成度 |
|------|------|------|--------|
| 基础学习 | 1-2 周 | 学习官方文档、理解核心概念 | 100% |
| 深入学习 | 2-3 周 | 学习高级特性、实践操作 | 100% |
| 模拟考试 | 1 周 | 参加模拟考试、分析错题 | 100% |
| 复习冲刺 | 3-5 天 | 复习重点内容、调整心态 | 100% |

### 1.2 学习方法

#### 理论学习
- **官方文档**：最权威的学习资料
- **白皮书**：深入理解架构设计原则
- **最佳实践**：学习实际应用场景

#### 实践操作
- **搭建实验环境**：实际操作加深理解
- **完成实验任务**：巩固知识点
- **解决实际问题**：提升实战能力

#### 模拟考试
- **定期参加**：检验学习成果
- **分析错题**：发现薄弱环节
- **查漏补缺**：有针对性复习

## 二、阿里云认证真题解析

### 2.1 计算服务

#### 题目1：ECS 实例选型
**题目：**
某公司需要部署一个 Web 应用，日均 PV 100 万，峰值 QPS 500，响应时间要求 < 100ms。应该选择哪种 ECS 实例类型？

**选项：**
A. 通用型 g6
B. 计算优化型 c6
C. 内存优化型 r6
D. 高主频型 hfg6

**正确答案：B**

**解析：**
- Web 应用主要消耗 CPU 资源
- 计算优化型 c6 适合计算密集型应用
- g6 是通用型，性价比一般
- r6 是内存优化型，适合数据库
- hfg6 是高主频型，适合游戏

**知识点：**
- ECS 实例类型：通用型、计算优化型、内存优化型、高主频型
- 应用场景匹配
- 性能需求分析

#### 题目2：弹性伸缩
**题目：**
某电商网站在促销期间流量激增，需要自动扩容。应该如何配置弹性伸缩？

**选项：**
A. 基于时间配置扩容规则
B. 基于监控指标配置扩容规则
C. 基于手动扩容
D. 基于固定实例数

**正确答案：B**

**解析：**
- 基于监控指标可以实时响应流量变化
- 基于时间不够灵活
- 手动扩容效率低
- 固定实例数无法弹性伸缩

**知识点：**
- 弹性伸缩的触发条件
- 监控指标：CPU 利用率、内存利用率、QPS
- 伸缩策略：扩容、缩容、冷却时间

### 2.2 存储服务

#### 题目3：OSS 存储类型
**题目：**
某公司需要存储大量的历史日志数据，访问频率低，但需要长期保存。应该选择哪种 OSS 存储类型？

**选项：**
A. 标准存储
B. 低频访问存储
C. 归档存储
D. 冷归档存储

**正确答案：D**

**解析：**
- 历史日志数据访问频率低，适合冷归档存储
- 冷归档存储成本最低
- 需要长期保存

**知识点：**
- OSS 存储类型：标准存储、低频访问存储、归档存储、冷归档存储
- 适用场景：热数据、温数据、冷数据
- 成本对比：冷归档 < 归档 < 低频访问 < 标准

#### 题目4：OSS 生命周期管理
**题目：**
某公司需要自动管理 OSS 数据生命周期，30 天后转为归档存储，180 天后删除。应该如何配置？

**选项：**
A. 使用 OSS SDK 手动管理
B. 配置生命周期规则
C. 使用定时任务
D. 使用 OSS 同步工具

**正确答案：B**

**解析：**
- OSS 生命周期规则可以自动管理数据
- 自动转换存储类型
- 自动删除过期数据

**知识点：**
- 生命周期规则配置
- 存储类型转换
- 自动删除策略

### 2.3 数据库服务

#### 题目5：RDS 读写分离
**题目：**
某电商网站读写比例 10:1，需要进行读写分离。应该使用什么方案？

**选项：**
A. 分库分表
B. RDS 读写分离
C. 使用缓存
D. 使用 MongoDB

**正确答案：B**

**解析：**
- RDS 读写分离适合读写比例高的场景
- 配置简单，性能提升明显
- 分库分表适合数据量大
- 缓存可以辅助但不能替代

**知识点：**
- RDS 读写分离原理
- 读写比例评估
- 只读实例配置

#### 题目6：数据库选型
**题目：**
某社交应用需要存储用户动态，数据量大，需要高并发写入。应该选择什么数据库？

**选项：**
A. MySQL
B. PostgreSQL
C. Redis
D. MongoDB

**正确答案：D**

**解析：**
- 用户动态数据结构灵活
- 需要高并发写入
- MongoDB 适合文档型数据
- MySQL/PostgreSQL 关系型数据库不适合

**知识点：**
- 数据库类型选择
- 关系型 vs 非关系型
- 应用场景匹配

## 六、考试技巧

### 6.1 答题技巧

#### 1. 仔细阅读题目
- 逐字逐句阅读
- 理解题目要求
- 注意关键词

#### 2. 识别关键词
```
关键词：
- most cost-effective（最成本效益）
- least expensive（最便宜）
- minimum（最少）
- maximum（最多）
- ensure（确保）
- requirement（要求）
- best practice（最佳实践）
```

#### 3. 排除法
- 排除明显错误的选项
- 缩小选择范围
- 提高正确率

#### 4. 管理时间
- 每题约 2 分钟
- 不确定的题目先标记
- 最后回来检查

### 6.2 常见陷阱

#### 陷阱1：过度设计
- 题目要求简单，选项复杂
- 选择最简单、最合适的方案

#### 陷阱2：成本优先
- 题目强调成本，选择成本最低的方案
- 注意关键词 "most cost-effective"

#### 陷阱3：可用性优先
- 题目强调高可用，选择多可用区方案
- 注意关键词 "highly available"

#### 陷阱4：性能优先
- 题目强调性能，选择高性能方案
- 注意关键词 "best performance"
